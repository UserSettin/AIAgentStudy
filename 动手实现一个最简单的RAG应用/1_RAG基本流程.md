# 1. RAG基本流程

## 整体流程图

```
用户提问 → 检索相关文档 → 构建Prompt → 大模型生成回答 → 返回结果
```

## 详细步骤

### 步骤1：文档准备与分块

将原始文档加载并切分成小块，便于后续向量化和检索。

- **加载文档**：从文件系统、数据库或网络加载文档
- **文本分块**：将长文档切分成合适大小的文本块
- **添加元数据**：为每个文本块添加来源、标题等元信息

### 步骤2：向量化与存储

将文本块转换为向量并存储到向量数据库中。

- **文本向量化**：使用嵌入模型将文本转换为向量表示
- **向量存储**：将向量和对应的文本块存储到向量数据库
- **建立索引**：为向量数据库建立索引，加速检索

### 步骤3：检索相关文档

根据用户问题，从向量数据库中检索最相关的文档。

- **问题向量化**：将用户问题转换为向量
- **相似度计算**：计算问题向量与数据库中所有向量的相似度
- **返回结果**：返回最相似的top-k个文档片段

### 步骤4：构建Prompt

将检索到的文档和用户问题组合成完整的Prompt。

- **模板设计**：设计合理的Prompt模板
- **上下文注入**：将检索到的文档作为上下文注入Prompt
- **问题整合**：将用户问题与上下文结合

### 步骤5：生成回答

使用大语言模型基于Prompt生成最终回答。

- **调用API**：调用大模型API
- **生成文本**：模型基于上下文生成回答
- **返回结果**：将回答返回给用户
