# 0. 什么是RAG

## RAG定义

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的AI技术。它通过以下方式增强大语言模型的能力：

1. **检索阶段**：从外部知识库中查找与用户问题相关的信息
2. **生成阶段**：基于检索到的信息，让大语言模型生成准确、可靠的回答

## RAG解决的问题

- **知识时效性**：大语言模型的训练数据是静态的，RAG可以引入最新信息
- **领域专业性**：可以针对特定领域构建知识库，提供专业回答
- **回答可追溯**：回答基于具体的文档内容，便于验证和溯源
- **减少幻觉**：基于真实文档生成回答，减少模型编造信息的可能

## RAG的核心优势

| 优势 | 说明 |
|------|------|
| 无需重新训练 | 不需要重新训练大模型，通过更新知识库即可扩展能力 |
| 灵活可控 | 可以精确控制模型回答的依据来源 |
| 成本效益 | 相比微调大模型，RAG的实现成本更低 |
| 透明可解释 | 可以追踪每个回答的来源文档 |

## 本教程目标

通过动手实现一个最简单的RAG应用，理解RAG的核心流程和关键技术点。
